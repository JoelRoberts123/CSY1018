Glossary of web dev terms

Alternative text - text that can be used to tell viewers the context of an image

Attribute - defines the property of a piece of code

Absolute positioning - specifies the positioning of certain html elements

CSS - Stands for Cacading Style Sheets. used to give web pages design

Body Tags - these contain the vast majority of content on your HTML pages

box model - used as a design element, the box model wraps around any html and puts it in a box.

Breakpoints - these are used to change the layout of the website when viewing on another devoce or making the internet tab smaller

Browser - this is what viewers will view the website on

Chrome devtools - used to inspect/change elements on any html page.

class - variable used in coding eg. h1, h2 etc

Client side - a script tha runs in the viewers browser

Closing tag - this denotes the end of a piece of content

Codepen - online code editing website, great for creating new code as well as searching for code that already exists

Command line - used to type in commands that the application will then carry out.

Containers - these help the CSS code certain peices of text.

Debugging - breaking down code to see how it operates.

DNS - Stands for Domain System name, this converts IP adresses into domain names.

Doctype -states which type of HTML is going to be used for the webpage

External style sheet - CSS file written in a seperate document that can be linked to multiple documents.

File path - refers to a specific location in a file system

Floats - This property allows you to place certain text in a certain postion eg float:left will move all text that this tag is applied to to the left.

Footer - used at bottom of website pages, usually contains info such as contact details

Fonts - fonts define the style of the text you type. these can be used to make the website look nicer.

Font style - describes which style of font the creator wishes to have

Forms - used on websites for the user to performs tasks such as sign up or send feedback to the creator(s)

Flex - specifies components with a flexible length

Flexbox- used to accomodate different dcreen sizes

header - used for headings of a website, defined in CSS using the h1,h2 tags

Hover - effects used for html text that will hover whenever a user hovers the mouse over that specific text.

Href - a reference to a link from another folder.

HTML - Hypertext markup language, used to construct webpage.s

Hyperlink - this is a link on one webpage that when clicked, will take you to another webpage.

Icons - images used to make any website look nicer as well as fill in empty space.

ID - used to style specific elements.

img - refers to an image.

Java - another programming language used for making and coding websites.

Li - lists, used for menu's

Link - refers to a link to a webpage embedded in the coding

Landing page - This is the main pahe of a website and is the first page a user will see when entering the website.

Media queries - Uesd to adapt screen sizes eg phone screen vs desktop screen.

Navigation - refers to the way a user makes his way through the site, usually comes in the form of a navigation bar at the top of the page.

Opening tag - This denotes the start of a piece of content.

Pseudo Class - used to change specific states of a selected element eg :hover will change properties of html when the user hovers over it.

Root Directory - this is the folder that will contain all HTML and CSS files for a webpage

Selector types - used to detect elements that the user wishes to style

Server side - refers to scripts that are run on the server rather than the users browser

src - refers to the source of an image, can be a hyperlink or a link to an image located in a folder

Validator - this is used for checking HTML/CSS to make sure there are no errors

UI - This stands for User interface and is what the user sees and how he/she navigates the site.

ul- like the li tag, this also refers to a list of tags

Universal - universal selectors will change all tags within a specific type eg. div, h1 etc.

Visual viewport - this sets the best size for the devices screen
